## Where the state answers well

- **Descriptive metadata** (Q1 scope, Q4 change log, Q10 open questions): high confidence; the v2.1 JSON carries explicit fields (`metadata`, `v2_1_changes_summary`, `source_owner_questions_open`) that make these direct lookups.
- **Self-aware semantics** (Q6 missing cost, Q11 SctLot, Q15 roadmap): the agent context and quality report give the harness exactly the language an answer should use, including the `missing != 0` rule and the SctLot reinterpretation.
- **Refusals** (Q7 org-wide): the state's caveat list and agent context's hard-limits section make refusal automatic; the harness returns `confidence='refused'` and `cannot_conclude=True`.
- **Coverage analytics** (Q2 strong, Q3 no-GL, Q14 ClickUp): per-project actuals and the `data_quality` block give clean numbers; the answer enumerates explicitly.
- **Trust boundaries** (Q8 lot grain, Q9 project/phase grain): the agent context's 5 enforced rules + the `vf_actual_cost_join_key` constant on every lot mean the harness can answer cleanly.

## Where answers are limited

- Per-lot dollar lookups are **inferred** by design (decoder-derived). The harness reports them as such — not a defect, but a label consumers must respect.
- Numerical questions about specific lots / phases (e.g. 'what is BCPD's actual cost for Parkway Fields Phase B1 Lot 112?') are not in the 15-question set; the harness's deterministic handlers do not compute these. They are demonstrated in `state_query_examples_v2_1_bcpd.md`.
- The harness does NOT validate that decoder-derived lot dollars match Lot Data; it inherits the W3 simulation results. Questions like 'show me a Harmony lot whose VF cost contradicts Lot Data' would require additional retrieval logic.
- Q11 (SctLot interpretation) and Q15 (data improvements) are inherently subjective; the harness states the v2.1 reading rather than ranking competing interpretations.

## Guardrails triggered

**None.** All 15 answers passed every applicable guardrail. This is the expected outcome — the harness was built specifically to reflect the agent context's enforcement rules.

Full guardrail rule list:

- **missing_cost_is_not_zero** — Missing cost is missing, not zero.
- **org_wide_unavailable** — Org-wide v2 is unavailable.
- **decoder_inferred** — Decoder-derived mappings are inferred unless source-owner-validated.
- **range_not_lot_level** — Range rows / shell allocations are not lot-level costs.
- **harmco_commercial_not_residential** — HarmCo commercial parcels are not residential lots.
- **harmony_3tuple_required** — Harmony joins require project + phase + lot.
- **sctlot_is_scattered_lots** — SctLot is Scattered Lots, not Scarlet Ridge.
- **qb_register_tieout_only** — QB register is tie-out only unless explicitly supported.
- **vf_is_one_sided** — Vertical Financials is cost-basis / asset-side, not full trial balance.
- **datarails_dedup_required** — DataRails 38-col raw sums are unsafe unless deduped.
